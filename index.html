4. Data Structure Recommendations

Folder Naming Conventions

Primary Folders

##_FOLDER_NAME_GREEK

Two-digit prefix for ordering, uppercase with underscores

Subfolders

##_category_name

Two-digit prefix, lowercase with underscores

Archive Folders

YYYY_archived_content

Year prefix for chronological organization

File Naming Templates

Documents

TYPE_THEME_YYYY_vX.ext

Example: LAW_ADOPTION_2024_v2.pdf

Forms

FORM_PURPOSE_YYYY_vX.ext

Example: FORM_FOSTER_CARE_APPLICATION_2024_v1.docx

Reports

REPORT_TYPE_YYYY_MM_vX.ext

Example: REPORT_FACILITY_INSPECTION_2024_03_v1.xlsx

Training Materials

TRAINING_TOPIC_YYYY_vX.ext

Example: TRAINING_CHILD_PROTECTION_2024_v3.pptx

Metadata Standards

Required Fields

  • • Document Title
  • • Creation Date
  • • Last Modified
  • • Author/Department
  • • Document Type
  • • Version Number

Optional Fields

  • • Keywords/Tags
  • • Subject Categories
  • • Related Documents
  • • Review Date
  • • Access Level
  • • Geographic Scope

System Fields

  • • File Size
  • • Download Count
  • • Last Accessed
  • • Checksum/Hash
  • • Backup Status
  • • Index Status

Version Control Strategy

Version Numbering

v1.0 Initial release
v1.1 Minor updates
v2.0 Major revision
v2.0_DRAFT Work in progress

Archive Policy

  • • Keep last 3 versions active
  • • Archive older versions annually
  • • Maintain legal documents indefinitely
  • • Compress archived content
  • • Create backup checksums
  • • Document version change reasons
